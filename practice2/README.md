(сортировка Пузырьком, Выбором и вставкой) на CPU с использованием OpenMP.

## Цель работы
Целью данной практической работы является:
- Освоить параллельное программирование с использованием OpenMP на CPU;
- Изучить и реализовать базовые алгоритмы сортировки (пузырьком, выбором, вставкой) в параллельном варианте.;
- Провести исследование производительности и сравнить параллельные и последовательные реализации.

---

## Реализованные алгоритмы

### Сортировка Пузырьком
Сортировка пузырьком работает путем последовательного сравнения соседних элементов массива и их обмена, если они расположены в неправильном порядке.  
В параллельной версии используется подход odd-even (по фазам), позволяющий избежать конфликтов между потоками.

### Сортировка Выбором
Сортировка выбором на каждом шаге находит минимальный элемент в неотсортированной части массива и помещает его в начало.  
В параллельной версии поиск минимального элемента выполняется в нескольких потоках с использованием OpenMP, а доступ к общей переменной защищается критической секцией.

### Сортировка Вставкой
Сортировка вставкой поэтапно формирует отсортированную часть массива, вставляя каждый новый элемент на правильное место.  
Из-за жесткой зависимости между итерациями эффективная параллелизация невозможна, поэтому в параллельной версии используется последовательная реализация.

## Анализ результатов
- Для малых массивов параллельные версии алгоритмов работают значительно медленнее из-за накладных расходов на создание и синхронизацию потоков.
- Для сортировки пузырьком наблюдается ускорение на больших массивах при параллельном выполнении.
- Параллельная сортировка выбором показывает худшую производительность из-за частого использования критических секций.
- Для сортировки вставкой параллельная версия практически не отличается от последовательной из-за зависимостей между итерациями.

---

## Вывод
Эксперимент показал, что параллельная реализация простых алгоритмов сортировки не всегда приводит к ускорению.  
Параллельное выполнение эффективно только для больших объёмов данных и при корректной организации алгоритма, тогда как для малых массивов и алгоритмов с сильными зависимостями последовательная реализация является более эффективной.
